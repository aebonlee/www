## 3. 프로젝트 아키텍처 패턴

### Context 구조 (공통)
```
App.jsx
├── ThemeProvider (다크/라이트/5색)
│   ├── LanguageProvider (ko/en)
│   │   ├── AuthProvider (Supabase Auth)
│   │   │   ├── ToastProvider (알림)
│   │   │   │   └── BrowserRouter
│   │   │   │       └── PublicLayout (Navbar + Routes + Footer)
```

### 파일 구조 (공통)
```
project/
├── public/
│   ├── 404.html          # SPA 리다이렉트
│   ├── CNAME             # 커스텀 도메인
│   └── favicon.svg
├── src/
│   ├── components/       # 공통 컴포넌트
│   │   └── layout/       # Navbar, Footer
│   ├── config/           # admin.js, site.js
│   ├── contexts/         # Theme, Language, Auth, Toast
│   ├── layouts/          # PublicLayout
│   ├── pages/            # 각 페이지
│   ├── styles/           # CSS 모듈
│   ├── utils/            # supabase.js, auth.js
│   ├── App.jsx
│   └── main.jsx
├── .env                  # 환경변수 (gitignore)
├── package.json
└── vite.config.js
```

### Supabase 테이블 패턴
```sql
-- 공통: user_profiles (모든 프로젝트 공유)
-- 프로젝트별: {project}_board_posts, {project}_board_comments
-- RLS: 모든 테이블 Row Level Security 적용
-- 트리거: handle_new_user (자동 프로필 생성)
```

## 5. 배포 플로우

```
개발자 PC
    ↓ npm run build
dist/ 폴더 생성
    ↓ npx gh-pages -d dist
GitHub Pages (gh-pages 브랜치)
    ↓ CNAME → DNS
*.dreamitbiz.com 접속 가능
```

**주의사항**:
- `.env` 파일은 각 개발 머신에 별도 생성 필요 (gitignore)
- GitHub Secrets는 GitHub Actions 전용 (로컬 빌드에 영향 없음)
- 빌드 시 환경변수가 JS에 인라인됨 (VITE_ 접두사 필수)

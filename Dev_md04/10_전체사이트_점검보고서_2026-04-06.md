## 1. 인증(Auth) 시스템 전면 수정

### 1-1. 발견된 문제
- **Google 로그인 불가**: 전체 사이트에서 Google OAuth 작동 안 함
- **이메일 회원가입 불가**: "Database error saving new user" 발생
- **기존 Kakao 사용자만 로그인 가능**: localStorage persistSession으로 세션 복원만 가능

### 1-2. 원인 분석 (3가지 독립 이슈)

| # | 원인 | 영향 범위 |
|---|------|----------|
| 1 | Google Cloud Console OAuth가 **테스트 모드** (0명 테스트 사용자) | Google 로그인 전체 불가 |
| 2 | 7개 사이트 OAuth **redirectTo URL 오류** (`origin + pathname` 사용) | ai-hub, allthat, papers, marketing, edu-hub, ai-data, ai-prompt |
| 3 | DB 트리거 **`handle_new_user`** 오류 (테이블명 + role 제약조건) | 모든 신규 가입 불가 |

### 1-3. 수정 내역

#### (A) Google OAuth 프로덕션 전환
- Google Cloud Console → OAuth 동의 화면 → "프로덕션 단계"로 전환

#### (B) 7개 사이트 OAuth redirectTo 수정

| 사이트 | 파일 | 변경 내용 |
|--------|------|----------|
| ai-hub | `src/utils/auth.js` | `origin + pathname` → `origin + '/'` |
| allthat | `src/utils/auth.js` | 동일 |
| papers | `src/utils/auth.js` | `origin` → `origin + '/'` (trailing slash 추가) |
| marketing | `src/utils/auth.js` | `origin + pathname` → `origin + '/'` |
| edu-hub | `src/utils/auth.js` | `origin + pathname` → `origin + '/'` |
| ai-data | `src/contexts/AuthContext.tsx` | `origin + '/login'` → `origin + '/'` |
| ai-prompt | `src/contexts/AuthContext.tsx` | `origin + pathname` → `origin + '/'` |

- 모든 사이트에 `emailRedirectTo: REDIRECT_URL` 추가

#### (C) DB 트리거 수정 (Supabase SQL)

**트리거 함수 4개 수정:**

```sql
-- 1. handle_new_user: profiles → user_profiles 테이블명 수정 + role='user' 명시
CREATE OR REPLACE FUNCTION handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.user_profiles (id, email, display_name, role, provider, signup_domain)
  VALUES (
    NEW.id, NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'full_name', NEW.raw_user_meta_data->>'name', split_part(NEW.email, '@', 1)),
    'user',
    COALESCE(NEW.raw_app_meta_data->>'provider', 'email'),
    COALESCE(NEW.raw_user_meta_data->>'signup_domain', '')
  );
  RETURN NEW;
EXCEPTION WHEN OTHERS THEN
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- 2~4. handle_agent_new_user, handle_forjob_new_user, handle_plan_new_user
-- 모두 EXCEPTION WHEN OTHERS THEN RETURN NEW 추가
```

**role DEFAULT 수정:**
```sql
ALTER TABLE user_profiles ALTER COLUMN role SET DEFAULT 'user';
-- 기존: DEFAULT 'member' (CHECK 제약조건 위반)
-- 변경: DEFAULT 'user' (CHECK: user, admin, superadmin, evaluator)
```

**누락 프로필 일괄 생성:**
```sql
INSERT INTO user_profiles (id, email, display_name, role, provider, signup_domain)
SELECT id, email,
  COALESCE(raw_user_meta_data->>'full_name', raw_user_meta_data->>'name', split_part(email, '@', 1)),
  'user',
  COALESCE(raw_app_meta_data->>'provider', 'email'),
  COALESCE(raw_user_meta_data->>'signup_domain', '')
FROM auth.users
WHERE id NOT IN (SELECT id FROM user_profiles)
ON CONFLICT (id) DO NOTHING;
```

### 5-1. Next.js i18n 루트 리다이렉트 수정 (3개 사이트)

**문제**: `output: 'export'` 정적 빌드에서 서버 `redirect()` 미작동 → 루트(/) 404

**수정 파일**:
- `D:\dreamit-web\books\src\app\page.tsx`
- `D:\dreamit-web\pbi\src\app\page.tsx`
- `D:\dreamit-web\pbirobot\src\app\page.tsx`

**수정 내용**: 서버 redirect → 클라이언트 `window.location.replace('/ko')` + meta refresh fallback

```tsx
'use client';
import { useEffect } from 'react';

export default function RootPage() {
  useEffect(() => {
    window.location.replace('/ko');
  }, []);

  return (
    <div>
      <meta httpEquiv="refresh" content="0; url=/ko/" />
      <p>Redirecting... <a href="/ko/">Click here</a></p>
    </div>
  );
}
```

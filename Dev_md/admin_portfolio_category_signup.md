# 관리자 대시보드 고도화 — 카테고리 SSOT · 사이트 포트폴리오 · 가입경로 분석

> 작업일: 2026-06-26

## 1. 목표

"종합관리 사이트"로서 120개 사이트 포트폴리오를 관리자 대시보드 전면에 드러내고, 사이트를 카테고리로 분류해 **가입경로(가입 출처)를 분석**한다. 메인(공개) 페이지 리디자인은 별도/추후.

## 2. 카테고리 SSOT 승격 (`siteRegistry.ts`)

- `SiteEntry`에 **`category: string`** 필드 추가. 120개 사이트 전부에 카테고리 주입(기존 AdminLinks의 19개 분류를 SSOT로 이관).
- **`SITE_CATEGORIES`**(라벨·색상·표시순서, 19개) + **`CATEGORY_COLOR`** 맵 + **`getCategoryCounts()`**(카테고리별 사이트 수, 빈 카테고리 제외) export 추가.
- `AdminLinks.tsx`: 자체 보유하던 카테고리·멤버십 목록 제거 → **레지스트리에서 파생**(`SITES.filter(category)`). 이제 카테고리/분류 변경도 siteRegistry 한 곳만 수정.

→ 카테고리 분류: Hub(9)·프로그래밍(6)·IT/CS(8)·AI교육(8)·AI도구(4)·비즈니스(4)·어학(3)·자격증(2)·핵심플랫폼(7)·출판문서(5)·취업경력(3)·개인포트폴리오(4)·로봇IoT(3)·강사(1)·연구학술(3)·클라우드(1)·학습법안전(3)·대학기업납품(13)·2026신규(33) = **120개**.

## 3. 가입경로 분석 (`adminStorage.ts`)

- **`getCategorySignups()`** 추가 — 회원의 `signup_domain`(없으면 `visited_sites[0]`)을 `getSite()`로 레지스트리 카테고리에 매핑해 집계. 레지스트리에 없는 출처(localhost 등)는 `기타/미분류`로 합산. 카테고리별 가입자 수 내림차순 반환.

## 4. 메인 대시보드 (`AdminDashboard.tsx`)

- **ZONE 1.5 「사이트 포트폴리오 + 가입경로」** 신설(환영 헤더 바로 아래, 커머스 통계 위).
  - 좌: 사이트 포트폴리오 — 전체 사이트(120)·결제 사이트·카테고리 수 미니스탯 + **카테고리별 사이트 분포** 가로 막대(레지스트리 파생, 동기). 「사이트 현황 →」 링크.
  - 우: **가입경로 분석** — 카테고리별 가입자 가로 막대(수·비율), 총 가입자 수 표시. (비동기 로드, 로딩/빈 상태 처리)
- 아이콘 globe·layers·route 추가.

## 5. 사이트 종합 현황 (`AdminSites.tsx`)

- **카테고리 열** 추가(색상 점 + 라벨 pill, 클릭 시 해당 카테고리 필터 적용).
- **카테고리 필터** 드롭다운(전체 + 19개) 추가, `filtered`에 반영.
- CSV 내보내기 헤더/행에 카테고리 컬럼 추가.

## 6. 사이드바 (`AdminSidebar.tsx`)

- '사이트 현황'·'사이트 바로가기' 메뉴에 **사이트 수 배지**(`SITES.length`) 추가 — 동적 신규(+N) 배지와 구분되는 중립 pill.

## 7. 인벤토리 메모

- GitHub aebonlee 공개 repo는 951개지만 대부분 dreamitbiz 사이트가 아님. 실제 사이트 = 레지스트리 120개(사용자 추정 "~122"와 일치). 레지스트리를 인벤토리 SSOT로 확정. 향후 추가 사이트는 siteRegistry에만 넣으면 전 페이지 자동 반영.

## 8. 검증

- `vite build` 성공(844 모듈). 카테고리 정합성: 120개 전부 유효 분류, 합계 120, 무효 참조 0.
- `.env`(VITE_IMP_CODE·VITE_PG_PROVIDER) 포함 빌드 → 결제 청크에 PortOne 값 정상 인라인 확인.
- ⚠️ 관리자 화면은 AdminGuard 인증 필요로 로컬 시각검증 미실시 — 빌드·데이터·패턴 일치로 검증.

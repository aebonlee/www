### 계정 상태 체계

| 상태 | DB 값 | 뱃지 색상 | 설명 |

| 차단 | `banned` | red | 영구 차단 |
| 탈퇴 | `deleted` | gray | 소프트 삭제 (기록 보존) |

## 세션 5: 회원가입 시 role/signup_domain 자동 설정

### 작업 배경

신규 가입 시 `role`이 기본값 `'user'`로 생성되고, `signup_domain`도 미설정. 가입 즉시 `'member'` + 접속 도메인이 자동 반영되도록 수정.

### 변경 내용

#### DB 트리거 업데이트 (`handle_new_user`)
- `role = 'member'` 명시 설정
- `signup_domain = raw_user_meta_data->>'signup_domain'` 읽기 (이메일 가입 시)
- `ALTER TABLE user_profiles ALTER COLUMN role SET DEFAULT 'member'`

#### `auth.js` — 이메일 가입 시 도메인 전달
- `signUp()` 함수에 `signup_domain: window.location.hostname` 메타데이터 추가

#### `AuthContext.jsx` — 기존 유저 자동 보정
- `loadProfile`에서 `signup_domain` 미설정 또는 `role`이 `'user'`인 경우 자동 수정
- OAuth(Google/Kakao) 가입 시에도 첫 로그인 때 자동 보정

### 수정 파일 (4개)

| 파일 | 변경 |
|------|------|
| `Dev_md/supabase_admin_functions.sql` | 트리거 업데이트 + role 기본값 변경 |
| `src/utils/auth.js` | signUp에 signup_domain 메타데이터 추가 |
| `src/contexts/AuthContext.jsx` | loadProfile에서 role/signup_domain 자동 보정 |

### 사용자 수동 작업

Supabase SQL Editor에서 실행 (supabase_admin_functions.sql 상단 섹션 0, 0-1):

## 세션 6: signup_domain 자동 설정 — check_user_status RPC 경유

### 작업 배경

OAuth(Google/Kakao) 가입 시 `signup_domain`이 자동으로 설정되지 않는 문제.
- 이메일 가입: `auth.js signUp()`에서 메타데이터로 전달 → 트리거에서 설정 (동작 O)
- OAuth 가입: OAuth 플로우에서 메타데이터 전달 불가 → 트리거에서 NULL 설정
- `loadProfile` 폴백: `updateProfile()`로 직접 UPDATE → RLS에 의해 차단 가능 (실패 무시)

### 해결 방안

이미 로그인마다 호출되는 `check_user_status` RPC (SECURITY DEFINER)에 `current_domain` 파라미터를 추가하여, `signup_domain`이 미설정인 경우 자동으로 설정. SECURITY DEFINER이므로 RLS를 우회.

### 변경 내용

| 항목 | 변경 |
|------|------|
| `check_user_status` SQL 함수 | `current_domain TEXT DEFAULT NULL` 파라미터 추가, signup_domain NULL이면 자동 설정 |
| `AuthContext.jsx` | `rpc('check_user_status')` 호출 시 `current_domain: window.location.hostname` 전달 |

### 수정 파일 (2개)

| 파일 | 변경 |
|------|------|
| `Dev_md/supabase_admin_functions.sql` | `check_user_status` 함수에 `current_domain` 파라미터 + signup_domain 자동 설정 로직 추가 |
| `src/contexts/AuthContext.jsx` | `check_user_status` RPC 호출 시 `current_domain` 전달 |

### 사용자 수동 작업

Supabase SQL Editor에서 `check_user_status` 함수 재생성 필요 (supabase_admin_functions.sql 섹션 5 실행):
```sql
CREATE OR REPLACE FUNCTION check_user_status(
  target_user_id UUID,
  current_domain TEXT DEFAULT NULL
) ...
```

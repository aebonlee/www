# 결제 플로우 상세 문서

## 전체 시퀀스

```
사용자 → Shop 페이지 → 상품 선택 → "장바구니 담기"
  ↓
장바구니 (Cart) → 수량 조절 / 삭제 → "결제하기"
  ↓
결제 (Checkout) → 주문자 정보 입력 → 결제수단 선택
  ↓
Supabase INSERT (orders + order_items, status: pending)
  ↓
PortOne.requestPayment() → KG이니시스 결제창
  ↓
사용자 결제 완료 → paymentId 반환
  ↓
verify-payment Edge Function 호출 (paymentId, orderId)
  ↓
PortOne REST API로 금액 검증
  ↓
검증 성공 → orders.payment_status = 'paid'
  ↓
/order-confirmation?orderId=xxx → 주문 완료 표시
```

## PortOne V2 SDK 사용법

```javascript
import * as PortOne from '@portone/browser-sdk/v2';

const response = await PortOne.requestPayment({
  storeId: 'store-xxxx',
  channelKey: 'channel-xxxx',
  paymentId: `payment-${orderId}`,
  orderName: 'DreamIT Biz 상품 N건',
  totalAmount: totalAmount,
  currency: 'CURRENCY_KRW',
  payMethod: 'CARD', // 또는 'TRANSFER'
  customer: {
    fullName: userName,
    email: userEmail,
    phoneNumber: userPhone
  }
});
```

## 결제 검증 (verify-payment)

```
1. 프론트에서 paymentId, orderId 전송
2. PortOne REST API 호출: GET /payments/{paymentId}
3. response.amount === DB orders.total_amount 확인
4. 일치 → payment_status = 'paid', paid_at = now()
5. 불일치 → payment_status = 'failed', 에러 반환
```

## 에러 처리

| 상황 | 처리 |
|------|------|
| 결제창 닫힘 | "결제가 취소되었습니다" 메시지 |
| 결제 실패 | "결제에 실패했습니다. 다시 시도해주세요" |
| 금액 불일치 | orders.status = 'failed', 관리자 알림 |
| 네트워크 오류 | "일시적 오류입니다. 잠시 후 다시 시도해주세요" |

## 보안 고려사항

- 결제 금액은 반드시 서버(Edge Function)에서 검증
- 프론트엔드 금액 조작 방지를 위해 DB 금액과 비교
- PortOne API Secret은 Edge Function 환경변수에만 저장
- .env는 .gitignore에 포함하여 커밋 금지

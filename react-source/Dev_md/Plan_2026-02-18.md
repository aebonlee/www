# DreamIT Biz 서비스 메뉴 재구조화 및 신규 페이지 추가 계획서

**작성일**: 2026-02-18 20:28 KST
**작성자**: Claude Opus 4.6
**프로젝트**: DreamIT Biz React SPA
**상태**: 구현 완료

---

## 1. 개요

DreamIT Biz React SPA의 Phase 1(완전 복원)이 완료된 상태에서, 기존 "서비스" 메뉴 하나에 6개 서비스가 평탄하게 나열되어 있던 구조를 분야별로 분리하고 신규 페이지를 추가한다.

### 변경 목표
- IT서비스, 교육, 출판을 각각 독립 상단 메뉴로 분리
- 블로그를 '커뮤니티'로 변경하여 블로그+게시판 통합
- 온라인 강의실, 전자출판/간행물/책, 게시판 신규 페이지 추가
- 프론트엔드 우선 구축 (백엔드는 추후)

---

## 2. 메뉴 구조 변경

### 변경 전
```
홈 | 서비스▼ | 포트폴리오▼ | 회사소개▼ | 블로그 | 연락처
     ├─ 웹개발
     ├─ 웹호스팅
     ├─ 디자인
     ├─ 컨설팅
     ├─ 기업 맞춤 강의
     └─ 출판사업
```

### 변경 후
```
홈 | IT서비스▼ | 교육▼ | 출판▼ | 포트폴리오▼ | 커뮤니티▼ | 회사소개▼ | 연락처
     ├─ 웹개발     ├─ 맞춤강의   ├─ 전자출판  ├─ 세종대     ├─ 블로그     ├─ 회사개요
     ├─ 웹호스팅   └─ 온라인강의실 ├─ 간행물   ├─ 진주교대   └─ 게시판     ├─ 비전
     ├─ 디자인                     └─ 책        └─ 프로젝트                  └─ 연혁
     └─ 컨설팅
```

## 4. 구현 단계

### Phase 1: 데이터 & 번역 (기반)
- `src/data/educationDetails.js` 생성 — 맞춤 강의 + 온라인 강의실 데이터
- `src/data/publishingDetails.js` 생성 — 전자출판 + 간행물 + 책 데이터
- `src/utils/translations.js` 확장 — nav, education, publishing, community 번역 키 추가

### Phase 2: CSS (스타일 선행)
- `src/styles/education.css` 생성 — 카테고리 카드, 강좌 카드, 게시판, 포맷 카드 등
- `src/index.css` — education.css import 추가
- `src/styles/navbar.css` — nav-link 패딩 축소 (20px→14px, 8개 항목 수용)
- `src/styles/responsive.css` — 1100px 브레이크포인트 추가 (8개 메뉴→모바일 전환)
- `src/styles/dark-mode.css` — 신규 컴포넌트 다크모드 추가

### Phase 3: 신규 페이지 (8개)
| 파일 | 내용 |
|------|------|
| `pages/Education.jsx` | 교육 랜딩 — 맞춤강의/온라인강의실 카테고리 카드 |
| `pages/EducationCustom.jsx` | 맞춤 강의 상세 — 기존 education 데이터 재활용 |
| `pages/Classroom.jsx` | 온라인 강의실 — 특장점 + 샘플 강좌 카드 |
| `pages/Publishing.jsx` | 출판 랜딩 — 전자출판/간행물/책 카테고리 카드 |
| `pages/PublishingEbook.jsx` | 전자출판 — 특장점 + 지원 포맷 |
| `pages/PublishingPeriodical.jsx` | 간행물 — 특장점 + 간행물 유형 |
| `pages/PublishingBook.jsx` | 책 — 특장점 + 도서 목록 + 프로세스 |
| `pages/Board.jsx` | 게시판 — 카테고리 필터 + 샘플 게시글 테이블 |

### Phase 4: 라우팅 & 네비게이션 연결
- `App.jsx` — 8개 lazy import 추가, 16개 라우트, /blog 리다이렉트
- `Navbar.jsx` — menuItems 8개 항목, isActive에 activePath 지원, 모바일 1100px 전환
- `serviceDetails.js` — education/publishing 키 제거 (4개만 유지)
- `ServiceDetail.jsx` — /services/education → /education 리다이렉트 추가
- `Services.jsx` — 4개 IT서비스만 표시, 제목 "IT Services"로 변경

### Phase 5: 홈 & 푸터 업데이트
- `Home.jsx` — education/publishing 카드 링크를 /education, /publishing으로 변경
- `Footer.jsx` — 퀵링크를 8개 메뉴 구조로 업데이트

### Phase 6: 빌드 & 문서화
- `npm run build` 정상 빌드 확인
- Dev_md 폴더에 계획서, README, CLAUDE.md 정리

```sql

CREATE TABLE board_posts (
  id BIGSERIAL PRIMARY KEY,
  category TEXT NOT NULL DEFAULT 'free',
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  author TEXT NOT NULL DEFAULT '익명',
  date TEXT,
  views INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- blog_posts (블로그)
CREATE TABLE blog_posts (
  id BIGSERIAL PRIMARY KEY,
  category TEXT, category_en TEXT,
  title TEXT NOT NULL, title_en TEXT,
  excerpt TEXT, excerpt_en TEXT,
  content TEXT, content_en TEXT,
  icon TEXT DEFAULT '📝',
  date TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- gallery_items (갤러리)
CREATE TABLE gallery_items (
  id BIGSERIAL PRIMARY KEY,
  title TEXT NOT NULL, title_en TEXT,
  category TEXT DEFAULT 'project',
  image_url TEXT,
  description TEXT, description_en TEXT,
  date TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## 향후 배포 프로세스 (교훈)

빌드 후 라이브 반영까지의 **필수 단계**:

```
1. cd react-source && npm run build      # Vite 빌드
2. 루트 assets/ 삭제                      # 이전 빌드 정리
3. react-source/dist/* → D:\www/ 복사     # 최신 빌드 배포
4. git add & commit & push               # GitHub Pages 반영
```

> `npm run build`만으로는 라이브 사이트에 반영되지 않음.
> 반드시 `dist/` → 루트 복사 + git push 필요.

## 기술 노트

- **Windows bash 호환성 이슈**: `cd "D:\www" && git status 2>&1 | head -20` 형태의 파이프 명령이 exit code 1 반환. `git -C D:/www` 문법으로 우회 해결.
- **Git 추적 상태**: 로컬에서 삭제된 파일이 Git에서는 여전히 추적 중 (Working tree에 `D` 표시). `git checkout HEAD --`로 복원 후 `git rm -rf`로 정리 필요.

## 1. 문제 상황

GitHub Actions CI/CD 파이프라인에서 `npm install` 단계 실패:

```
npm error ERESOLVE could not resolve
npm error peer react@"^16.6.0 || ^17.0.0 || ^18.0.0" from react-helmet-async@2.0.5
npm error Found: react@19.2.4
```

**원인**: `react-helmet-async@2.0.5`의 peerDependency가 `react@^16 || ^17 || ^18`까지만 지원하며, 프로젝트가 사용 중인 React 19.2.x와 충돌. 패키지는 2년 이상 업데이트 없이 사실상 방치 상태.

## 2. 해결 방안 검토

| 방안 | 장점 | 단점 | 선택 |
|------|------|------|------|
| `--legacy-peer-deps` 플래그 | 즉시 해결 | 근본 해결 아님, CI마다 플래그 필요, 잠재 호환성 문제 | X |
| `@dr.pogodin/react-helmet` 포크 | React 19 지원 | 새 외부 의존성 추가, API 변경 가능성 | X |
| **React 19 네이티브 메타태그** | 의존성 제거, 번들 감소, 공식 지원 | SEOHead 컴포넌트 내부 수정 필요 | **O** |

**최종 결정**: React 19가 `<title>`, `<meta>`, `<link>` 태그를 컴포넌트 내에서 직접 렌더링하면 자동으로 `<head>`에 호이스팅하는 네이티브 기능을 지원하므로, 외부 라이브러리 없이 전환.

## 3. 구현 상세

### 3.1 SEOHead 컴포넌트 변환

**Before** (react-helmet-async):
```jsx
import { Helmet } from 'react-helmet-async';
// ...
return (
  <Helmet>
    <html lang={language} />
    <title>{fullTitle}</title>
    <meta name="description" content={desc} />
    ...
  </Helmet>
);
```

**After** (React 19 네이티브):
```jsx
// import 없음 — React 19 내장 기능
return (
  <>
    <title>{fullTitle}</title>
    <meta name="description" content={desc} />
    ...
  </>
);
```

- `Helmet` 래퍼 → React Fragment (`<>...</>`)로 변환
- `<html lang>` 태그는 React 19 네이티브에서 직접 지원하지 않으므로 제거 (LanguageContext에서 `document.documentElement.lang` 직접 설정으로 대체 가능)
- 나머지 `<title>`, `<meta>`, `<link>` 태그는 그대로 유지 — React 19가 자동 호이스팅

### 3.2 main.jsx HelmetProvider 제거

```jsx
// Before
import { HelmetProvider } from 'react-helmet-async'
<HelmetProvider><App /></HelmetProvider>

// After
<App />
```

### 3.3 package.json 의존성 삭제

```diff
- "react-helmet-async": "^2.0.5",
```

## 8. 참고 사항

- React 19의 메타태그 네이티브 호이스팅은 [React 19 공식 블로그](https://react.dev/blog/2024/12/05/react-19)에서 공식 기능으로 소개됨
- `react-helmet-async`는 2년 이상 업데이트 없이 React 19 지원 미정 상태 (GitHub Issue #239)
- 향후 `<html lang>` 동적 전환이 필요하면 `LanguageContext`에서 `document.documentElement.lang = language` 직접 설정 방식 권장

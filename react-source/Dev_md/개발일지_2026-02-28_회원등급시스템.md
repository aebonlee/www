## 1. 작업 개요

관리자 회원 관리 페이지(`/admin/users`)의 권한 체계를 2단계(관리자/일반)에서 4단계(관리자/VIP회원/정회원/준회원)로 확장하고, 관리자가 드롭다운으로 즉시 등급을 변경할 수 있도록 개선했다.

**Before**: 관리자/일반 2단계 텍스트 뱃지 (변경 불가)
**After**: 4단계 등급 드롭다운 + 등급별 필터 탭 + 즉시 DB 반영

## 3. 등급 체계

| 등급 | DB 값 | 뱃지/드롭다운 색상 | 설명 |
|------|--------|-------------------|------|
| 관리자 | `admin` | red (#FEE2E2) | ADMIN_EMAILS 기반 자동 + 수동 지정 |
| VIP회원 | `vip` | purple (#F3E8FF) | 관리자가 수동 지정 |
| 정회원 | `member` | blue (#DBEAFE) | 기본 등급 |
| 준회원 | `associate` | gray | 신규 가입 or 관리자가 수동 지정 |

## 4. 구현 상세

### 4.1 adminStorage.js — `updateUserRole(userId, role)`
- Supabase `user_profiles` 테이블의 `role` 컬럼을 직접 업데이트
- 에러 시 `{ error }` 반환, 성공 시 `{ success: true }` 반환

### 4.2 AdminUsers.jsx — 등급 드롭다운
- `ROLE_OPTIONS` 상수 배열로 4단계 등급 정의 (value, label, color)
- `resolveRole(user)` 함수: ADMIN_EMAILS 해당 유저는 무조건 `admin` 반환
- ADMIN_EMAILS 유저: `<span className="td-badge red">관리자</span>` 고정 표시 (드롭다운 아님)
- 그 외 유저: `<select className="role-select role-{role}">` 드롭다운 렌더링
- `handleRoleChange`: 낙관적 업데이트(optimistic update) → DB 반영 → 실패 시 롤백 + alert

### 4.3 AdminUsers.jsx — 등급 필터 탭
- `roleFilter` 상태 추가 (기존 `filter`, `siteFilter`와 독립)
- 필터 탭 순서: 등급 필터 → 가입방법 필터 → 가입 사이트 필터
- 각 등급별 카운트 표시, 0건 등급은 자동 숨김

### 4.4 admin.css — `.role-select` 스타일
- `appearance: none` + CSS 화살표 아이콘 (SVG data URI)
- `border-radius: var(--radius-full)` — 기존 뱃지와 동일한 pill 형태
- 등급별 배경/텍스트 색상: `.role-admin`(red), `.role-vip`(purple), `.role-member`(blue), `.role-associate`(gray)
- hover 시 border 표시, focus 시 box-shadow 포커스 링

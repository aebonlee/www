# 3차 개발: 이커머스 (장바구니 + 결제) 구현 계획

**프로젝트**: DreamIT Biz React SPA
**개발 단계**: 3차 개발
**작성일**: 2026-02-18
**담당**: Claude Opus 4.6

---

## 1. 개요

DreamIT Biz React SPA의 2차 개발(메뉴 재구조화)이 완료된 상태.
출판(책/간행물/전자출판)과 교육(강의) 상세 페이지에 가격 정보와 구매 기능을 추가.
장바구니에 담고 카드결제(KG이니시스) 또는 계좌이체로 구매할 수 있는 기능을 구현한다.

## 2. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 결제 SDK | PortOne V2 (`@portone/browser-sdk`) | 프론트엔드 결제창 호출 |
| PG사 | KG이니시스 | 실제 카드결제/계좌이체 처리 |
| DB | Supabase PostgreSQL | 상품, 주문, 주문상세 테이블 |
| 인증 | Supabase Auth | 구매자 회원가입/로그인 |
| 서버 | Supabase Edge Functions (Deno) | 결제검증, 웹훅, 주문처리 |
| 상태관리 | CartContext (React Context) | 장바구니 상태 (localStorage 영속) |

## 3. 구현 Phase별 상세

### Phase 4: Supabase 백엔드

### Phase 5: PortOne 결제 연동

## 7. 결제 플로우

```
1. 사용자: 장바구니 → 결제하기 클릭
2. Checkout: 주문정보 입력 (이름, 이메일, 전화번호)
3. Checkout: Supabase에 orders + order_items INSERT (pending)
4. Checkout: PortOne.requestPayment() 호출
5. 결제창 → 사용자 결제
6. 결제 완료 → paymentId 수신
7. Checkout: verify-payment Edge Function 호출
8. 검증 성공 → /order-confirmation?orderId=xxx
9. OrderConfirmation: 주문 상세 표시
```

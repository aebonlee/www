**문제**: 통계 카드의 금액 값(`135,000원`, `취소금액` 등)이 28px 폰트로 넘침
**해결**: `AdminStatCard`에서 문자열 값일 때 `stat-value-sm` 클래스 자동 적용

```css
.stat-value-sm { font-size: 18px; }          /* 기본 */
@media (max-width: 768px) { font-size: 15px; } /* 모바일 */
```

## 5. 카카오 로그인 안정성 개선

### 5.1 발견된 문제

1. **레이스 컨디션** — `getSession()`이 OAuth 코드 교환 완료 전에 `loading=false` 설정
2. **Supabase 클라이언트 auth 설정 누락** — `flowType`, `detectSessionInUrl` 미명시
3. **카카오 OAuth scopes 미지정** — 프로필/이메일 데이터 누락 가능
4. **signOut 전역 실패** — OAuth 세션 만료 시 에러 발생
5. **OAuth 콜백 에러 미감지** — Login 페이지에서 실패 시 에러 미표시

### 5.2 수정 내역

| 파일 | 변경 내용 |
|------|-----------|
| `supabase.js` | `createClient`에 `flowType: 'pkce'`, `detectSessionInUrl: true`, `autoRefreshToken: true`, `persistSession: true` 명시 |
| `AuthContext.jsx` | `getSession()` 제거 → `onAuthStateChange` `INITIAL_SESSION` 이벤트로 통합 + 5초 안전장치 타이머 |
| `auth.js` | 카카오 scopes: `profile_nickname profile_image account_email` 추가 |
| `auth.js` | `signOut({ scope: 'local' })` — OAuth 세션 만료 시 에러 방지 |
| `Login.jsx` | OAuth 콜백 URL `error`/`error_description` 파라미터 감지 + URL 자동 정리 |

### 5.3 핵심 원리

**변경 전:**
```
페이지 로드 → getSession() → null (코드 교환 미완료) → loading=false → "로그인 안 됨"
                                                      ↓
                                     onAuthStateChange SIGNED_IN → 너무 늦음
```

**변경 후:**
```
페이지 로드 → onAuthStateChange 등록
          ↓
Supabase 코드 교환 완료 → INITIAL_SESSION(session) → loading=false → "로그인 됨"
```
